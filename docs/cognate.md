# Thoth Cognitive Executive System — Architecture Summary
**Version:** 2.0 (Cognate Phases 1–12 Verified)

---

## 1. What the Cognitive Executive System Is

The Cognitive Executive System is the reasoning and orchestration layer that sits above individual tools and retrieval. It is what makes Thoth an **agent** rather than a chatbot.

Where GRAG answers the question *"what knowledge do I need right now?"*, the Cognitive Executive System answers the question *"what should I do, and in what order, and what do I do when things go wrong?"*

It is composed of five tightly integrated components:

- **Plan:** A structured, variable-length representation of a goal generated by the LLM.
- **IPlanner:** The interface responsible for creating and revising plans using dynamic LLM generation.
- **WorkflowEngine:** The execution harness that dispatches steps to the ToolRegistry or RAGPipeline.
- **ExecutiveController:** The central state machine that drives the loop from goal to completion.
- **StrategyEngine:** The learning layer that analyzes trajectories to optimize future reasoning.

These components form the **Cognitive Spine** of Thoth.

---

## 2. Current Implementation Status

The Cognitive Executive System has transitioned from a hardcoded scaffold to a fully dynamic, persistent, and resilient engine.

### ExecutiveController (Verified Stable)
The `ExecutiveController` is a resumable, thread-safe state machine. It manages the asynchronous execution loop and ensures the agent's state is consistent even across crashes.

**Confirmed States:**
`IDLE → PLANNING → EXECUTING_STEP → OBSERVING_RESULT → REVISING_PLAN → COMPLETED/FAILED`

**Verified Features:**
- **Full Resumability:** Entire execution state (plan, step index, results) is persisted to SQLite in real-time.
- **Thread-Safety:** Robust locking for multi-threaded GUI/Agent interaction.
- **GRAG Wiring:** Automatically calculates and syncs Goal (G) and State (C) embeddings.
- **Security:** Integrated with `ConstraintChecker` for global tool execution policy.

### WorkflowEngine (Operational)
The `WorkflowEngine` centralizes the dispatch of tasks. It manages:
- **Tool Dispatch:** Routing `TOOL` steps to the `ToolRegistry`.
- **Retrieval Dispatch:** Routing `RETRIEVAL` steps to the `RAGPipeline`.
- **Parallelism:** Support for batched tool execution (Parallel Engine v1.0).
- **Error Handling:** Enforcement of `StepFailurePolicy` (retries, aborts, or revisions).

### Advanced Reasoning Modes
Thoth supports pluggable execution strategies via the **Strategy Pattern**:
- **StandardExecutionMode:** The default path for sequential task completion.
- **ScientificExecutionMode:** A hypothesis-driven iterative loop. It uses a `ProblemState` to track hypotheses, constraints, and rejected paths, iterating until convergence.

### Experience-Aware Retrieval
Retrieved context is no longer static. It is informed by the agent's recent history:
- **TrajectoryBuilder:** Generates a semantic vector **T** from the last 7 steps (actions/failures).
- **Plan History Reuse:** Past successful plans are embedded and retrieved during the `PLANNING` phase to accelerate goal decomposition.

## 3. How the Cognitive Loop Works

One complete goal execution cycle follows this path:

1.  **Goal Ingestion:** User submits objective; `StrategyEngine` retrieves relevant historical strategy hints.
2.  **Dynamic Planning:** `IPlanner` generates a variable-length plan tailored to the goal.
3.  **State Evaluation:** Before each step, the controller updates the current state embedding **C** and calculates direction **D = G - C**.
4.  **Directional Retrieval:** GRAG retrieves context using the blend of Query, Direction, and Trajectory.
5.  **Execution:** `WorkflowEngine` dispatches the step to the appropriate mode/subsystem.
6.  **Observation & Learning:** The result is analyzed; `TrajectoryBuilder` updates the semantic context; the loop repeats or triggers a **Plan Revision** if the goal direction shifts.

---

## 4. Relationship to Other Systems

### GRAG (Retrieval)
GRAG is the "eyes" of the agent. The Executive System provides the "gaze" by orienting retrieval toward the active subgoal and away from past failures via the trajectory vector.

### Tool System
Tools are the "hands" of the agent. They are strictly encapsulated; the Executive System never knows how a tool works, only its input schema and its result status.

### NODE System
The NODE system provides a modular wiring layer. Eventually, the `WorkflowEngine` will treat complex Node Graphs as a single `NODE` step type, allowing for nested, deterministic sub-routines.

## 5. Future Frontiers (Not Yet Built)

While the "Spine" is verified, the following upgrades are on the immediate roadmap:

1.  **Hierarchical Goal Trees:** Transitioning from flat plan lists to nested `GoalNode` trees for massive, multi-day engineering tasks.
2.  **Adaptive Graph Learning:** Moving beyond the prototype graph memory to dynamic edge-weighting that learns from execution success.
3.  **Autonomous Tool Self-Building:** Leveraging the complete cognitive loop to allow Thoth to generate and test its own new tools.

---

## 6. Summary

Thoth is an **Experience-Aware Cognitive System**. It does not simply "prompt" an LLM; it manages a persistent state, learns from its trajectory, and navigates a large documentation space with directional intent.

The architecture is locked, the baseline is verified, and the system is ready for scale.
